# Agent Define · 定义阶段

引导用户完成 agent 的核心定义。每次只问一个问题，等用户回答再继续。不要一次抛多个问题。

## 引导序列（严格按顺序）

### 问题 0：会话模式（最先问）

> "这个 agent 是单次会话还是多次会话？
>
> A. 单次——用户每次对话完成一个任务，对话结束即完成
>    示例：量化策略验证官、皮克球成长教练
>
> B. 多次（记忆型）——agent 记住用户，每次对话基于历史累积
>    示例：就医向导（记住每次就诊）、学习伴侣（记住进度）"

接收后确认："选 A → '好，我们设计单次会话 agent。' 选 B → '好，我们设计记忆型 agent——DONE-WHEN 会有两层。'"

**记录会话模式，影响 Q7.5 和 Q8。**

---

### 问题 1：领域

> "你在哪个领域有深度经验？（工作/专业/长期研究的方向——越具体越好，不是'互联网'，是'B端SaaS产品增长'）"

接收回答后，总结一句话确认："你的领域是 [X]，对吗？"

---

### 问题 2：英雄原型

> "在这个领域里，有没有某个人物（真实或虚构）、某种专家视角、某套方法论——你觉得特别对？这个'英雄'的核心判断是什么？"

引导原则：
- 英雄不必是名人，可以是某种思维方式（"像审计师一样思考"）
- 要提炼出英雄的**核心判断**（不是简历，是世界观）
- 示例：Edward Thorp = "一切判断必须建立在可量化的概率优势之上，直觉不是证据"

**三层穿梭检验**（收到用户的英雄描述后，用这三层检验质量）：

| 层 | 内容 | 判断方式 |
|----|------|---------|
| 现象层 | 他是谁、做过什么——真实年份、作品、成就 | 能 Google 到的具体事件 |
| 本质层 | 他怎么思考——独家看世界的方式 | 和别人不同的那个角度 |
| 哲学层 | 他会说的铁律——最好是原话 | 一句话，能刻在墙上 |

三层全在 = 英雄可用。缺任何一层，追问补齐：
- 缺现象层："他具体做过什么让你印象深刻的事？有没有真实年份或作品？"
- 缺本质层："他看这个领域的独家角度是什么？跟普通专家有什么不一样？"
- 缺哲学层："他有没有一句会反复说的铁律或名言？"

---

### 问题 3：痛点（引导写 3 个）

> "用这个英雄的眼睛看你的领域，你的目标用户现在遭受哪 3 个真实的痛？"

每个痛点必须包含三要素，逐条引导：
1. **具体长什么样**（不是"很麻烦"，是"每次做 X 时，会出现 Y 现象"）
2. **为什么真实**（有没有数字/案例/你亲历的证据）
3. **现在怎么对付**（大家现在用什么蹩脚方案）

如果用户给的太模糊，追问："能给我一个你亲眼见过的具体场景吗？"

---

### 问题 4：选种子

列出 3 个痛点，问：

> "选一个作为你第一个 agent 的出发点——哪个痛点：
> A. 影响最根本（其他痛点是它的下游）
> B. 最容易定义'验证通过'的标准（不依赖用户主观判断）
> C. 你自己最有独家信息/方法论
> 
> 选哪个？说说为什么。"

接收后写一句话种子："一个 agent，帮 [WHO]，在 [WHEN]，做 [WHAT]，把 [人性弱点/执行难点] 从决策链里移除。"

---

### 问题 5：WHO

> "你的 agent 服务的人，具体是什么样的人？
> 
> 请描述：
> - 他们的身份/背景
> - 他们手上有什么（工具/数据/结果）
> - 他们站在什么节点上（准备做什么决定）
> - 他们通常有什么心理状态
> 
> 然后反面：**你不服务谁**？（明确排除，防止 agent 被滥用）"

---

### 问题 6：WHEN

> "你的 agent 在哪一刻启动？描述那个具体场景——用户打开这个 agent 之前的 30 秒在做什么？"

引导原则：越具体越好。"鼠标停在按钮上方"比"准备上线时"好。

---

### 问题 7：WHAT

> "这个 agent 要解决的具体的痛是什么？用一句话说清楚——不是功能列表，是'把 [主观判断/人性弱点] 变成 [客观标准/可量化结论]'。"

---

### 问题 7.5：平台能力选择

> "你的 agent 会用到哪些平台能力？（可多选，不选直接跳过）
>
> ☐ 语音输入——用户用说话代替打字
> ☐ 文件上传——用户上传文档/图片/化验单等
> ☐ 文件输出——agent 生成可下载的结果文件
> ☐ 超强记忆——agent 跨会话记住用户（仅 Q0=B 记忆型 agent 可选）"

针对勾选项追问：

**文件上传追问：**
> "用户会上传什么类型的文件（PDF/图片/Excel/…）？agent 要从中提取哪些关键信息？"

**文件输出追问：**
> "agent 要输出什么文件？文件名？用户拿到后用来做什么？"

（语音输入和超强记忆：无需追问，后续步骤自动激活对应引导）

没有勾选任何项 → 跳过，继续 Q8，流程不变。

---

### 问题 8：DONE-WHEN

**单次会话（Q0=A）：**
> "对话什么时候算结束？给我一个**可以被观测到的信号**——不是'用户满意了'，是'用户手里拿到了 X，并且说出了 Y'。"

引导原则：
- 必须是外部可观测的行为，不是情绪判断
- 要有两个条件（文档交付 + 明确下一步）

**记忆型 agent（Q0=B）的 DONE-WHEN 分两层：**
> "本次会话结束信号是什么？（用户拿到了什么？说出了什么？）
>
> 这次会话结束后，记忆里新增/更新了哪些信息，供下次会话使用？"

引导原则：
- 本次会话信号：同单次，外部可观测行为
- 记忆更新：具体字段名 + 写入时机（如"就诊日期 → 写入，下次展示"）

## 完成后输出

所有 8 个问题回答完毕后：

1. 汇总展示所有定义，请用户确认
2. 用户确认后，创建文件结构：
   ```
   agents/[agent-name]/
   ├── agent-script.md
   └── knowledge-base/
   ```
3. 写入 `agent-script.md` 第零部分

**写入格式：**

```markdown
# [Agent 名称]

**版本：** v0.1
**类别：** [用户选择或引导判断：A目标驱动闭环型 / B开放探索型 / C情绪陪伴型]
**英雄原型：** [人物名 + 一句话核心判断]

---

## 第零部分 · 核心定义
> 由 /agent-define 生成

**WHO · 服务的人：**
[特征描述]

**WHO · 不服务：**
[排除对象 + 原因]

**WHEN · 启动时刻：**
[具体场景]

**WHAT · 解决的痛：**
[一句话]

**DONE-WHEN · 结束信号：**
[可观测条件，含两个要素]
```

**如果用户选了任何平台能力（Q7.5 有勾选），在第零部分之后追加第零部分B：**

```markdown
## 第零部分B · 平台能力设计
> 由 /agent-define 生成

**会话模式：** 单次 / 多次（记忆型）

**选用能力：**
- 🎤 语音输入（如选）
- 📎 文件上传：用户上传 [X]，agent 提取 [Y]（如选）
- 📄 文件输出：输出 [文件名]，包含 [内容]（如选）
- 🧠 超强记忆：记忆槽见第四部分（如选）
```

未选任何平台能力 → 不写入第零部分B，agent-script.md 格式与现在完全一样。

4. 提示用户下一步：`/agent-system-prompt`

---

## 类别判断参考

| 类别 | 特征 |
|------|------|
| A — 目标驱动闭环型 | 有明确起点/终点/不可跳过的中间步骤 |
| B — 开放探索型 | 用户目标模糊，agent 帮助探索和发现 |
| C — 情绪陪伴型 | 以陪伴/共鸣为主，结构化程度低 |
