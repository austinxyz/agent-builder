# Agent Dialogue · 对话剧本

引导用户把 System Prompt 里的核心任务，变成可执行的对话步骤。

## 前置：读取文件

先读取 `agent-script.md` 中的第一部分（核心任务区块）。  
如果不存在，提示："请先运行 `/agent-system-prompt` 完成剧本。"

同时读取 `第零部分B · 平台能力设计`（如存在），记录选用的平台能力。后续引导将根据选用能力激活对应步骤类型。

**能力标记（后续步骤引导中使用）：**
- 有 🎤 语音输入 → 所有台词写完后触发语音友好审查
- 有 📎 文件上传 → 可选 📎 文件步骤类型
- 有 📄 文件输出 → 最后一步追加文件输出设计
- 有 🧠 超强记忆 → Step 1 前追加会话开始协议；最后一步追加记忆更新步骤；生成第四部分

展示核心任务步骤列表，告诉用户："我会把每个步骤展开成完整的对话剧本。"

---

## 步骤数确认

> "你的 agent 需要几步对话？（核心任务有几个步骤，通常对应几步对话）
> 
> 建议 4-6 步。太少会跳步，太多用户会疲惫。"

确认步骤数后，逐步引导，每步完成后确认再继续。

---

## 每一步的引导框架

对每个 Step N，依次引导填写：

**（如选了 📎 文件上传）首先确认此步骤是否需要收文件：**
> "这一步需要用户上传文件吗？如果是，启用 📎 文件步骤类型。"

选择文件步骤 → 用下方 📎 文件步骤框架替代标准 🎯/✅/🗣 框架。
不需要 → 继续使用标准框架（🎯/✅/🗣/📌/〔越界〕/📚）。

### 🎯 子目标

> "Step [N] 的子目标是什么？——这一步结束时，agent 拿到了什么？得出了什么结论？"

要求：一句话，动词开头，具体可量化。  
❌ "建立信任"  
✅ "拿到用户的 5 个基础数字，能进入闸门计算"

---

### ✅ 关闭准则

> "这一步什么时候算完成？给我一个可以被观测到的信号——不接受'用户满意了'。"

追问：
- "如果用户给出模糊答案（'差不多''大概''记不清'），算关闭吗？"
- "需要几个条件同时满足才能进入下一步？"

引导原则：
- 关闭准则是"拿到 X 信息"或"用户明确表示接受 Y 假设"
- 不是情绪判断，是行为判断

---

### 🗣 标准台词

> "agent 在这一步说什么？写出标准对话文本——用户看到的第一句话。"

引导原则：
- 先写核心问题，再写为什么问（帮用户理解，降低抗拒）
- 如果要收集多个信息，列成编号清单（不要用问答堆砌）
- 语气要跟英雄一致

---

### 📌 Fallback 路径

> "如果用户给出这些答案，agent 怎么办：
> 
> 1. 用户给出模糊答案（'记不清''差不多'）
> 2. 用户拒绝提供信息（'这有什么关系''直接告诉我结论'）
> 3. 用户的情况不符合这一步的预期"

对每种情况引导写：
- 第 1 轮追问（引导用户自己推算）
- 第 2 轮仍无结果 → **触发保守上限申报**（agent 主动声明假设，让用户确认）
- 用户接受假设 → 关闭此项，记录注明

关键设计原则（S5 核心洞察）：  
**关闭准则不能等"完美答案"，要能接受"用户明确同意保守假设"**  
→ agent 掌握推进权，用户无法用软答案绕过闸门

---

### 〔越界场景〕

> "这一步最可能出现哪种'越界'？用户会说什么？agent 怎么守边界？"

引导写至少 1 个越界场景，格式：
```
〔越界场景〕用户说："[原话]"
agent：[守边界的回应，不争论，不退让，引用铁律]
```

---

### 📚 KB 引用

> "这一步需要引用哪些专业知识/数据/案例来支撑判断？
> 
> 比如：统计数字、行业基准、真实案例、计算公式、代码模板"

列出引用点，格式：`knowledge-base/0N_[主题].md`  
（KB 文件由 `/agent-kb` 后续生成）

---

## 📎 文件步骤框架（选了文件上传且此步需要收文件时使用）

**替代**标准 🎯/✅/🗣 框架，填写以下字段：

### 🎯 子目标
> "这步文件步骤的子目标是什么？（格式：获取并解析用户上传的 [文件类型]，拿到 [关键信息]）"

### 📎 提取模板
> "agent 需要从文件里提取哪些信息？
>
> 必须提取（没有则无法继续）：[字段1] / [字段2]
> 有则提取（可选，补充判断）：[字段3]
> 忽略（噪声）：[其他信息]"

### ✅ 关闭准则
必须提取字段均已获取 → 继续下一步
文件质量不足 → 触发 Fallback B

### 🗣 标准台词
> "agent 如何引导用户上传文件？台词需包含：上传什么文件、上传到哪里、上传后 agent 会做什么"

### 📌 Fallback（文件专属三类）

| 情况 | Agent 动作 |
|------|-----------|
| A. 文件格式不对 | "请上传 [格式]，当前不支持 [X]" |
| B. 图片模糊/扫描歪 | "能否重拍？重点拍清楚 [关键区域]" |
| C. 用户没有文件 | "没有也可以——我来问你几个问题代替，效果稍减" |

---

## 🧠 记忆钩子（选了超强记忆时，在每步引导末尾追加）

> "这步结束时，需要记住什么供下次会话使用？
>
> 格式：存入字段 [名称]：[值]，下次在 Step [N] 读取"

如果本步无需存入记忆 → 填"无"，跳过。

---

### 🧠 会话开始协议（记忆型 agent 的 Step 1 固定前置，在 Step 1 之前引导）

引导用户写出 Step 1 开头的记忆读取逻辑：

```
🧠 会话开始协议：
有历史记忆：
  agent 台词："欢迎回来！上次 [日期] 我们 [X]。今天继续还是重新开始？"
  → 继续：读取记忆，从上次断点进入
  → 重新开始：保留记忆但走新用户流程

无历史记忆：
  agent 台词："[新用户欢迎语，直接进入 Step 1 正常流程]"
```

---

### 🧠 记忆更新步骤（记忆型 agent 最后一步固定追加）

引导用户在最后一步台词末尾追加：

```
🧠 记忆更新（本次会话结束）：
更新字段：[字段名: 新值]
新增字段：[字段名: 值]（如有）
结束台词含记忆确认："我记住了你的 [X]，下次继续。"
```

---

## 🎤 语音友好审查（选了语音输入时，所有步骤台词写完后统一执行一轮）

逐条审查每步的 🗣 标准台词，检查以下问题：

**不通过条件（必须改）：**
- ❌ Markdown 有序列表（`1. 2. 3.`）——口头无法区分
- ❌ 表格结构——口头说不清
- ❌ 超过 3 个并列项——口头难以跟随
- ❌ 代码块——语音无法传达

**改法指南：**

| 原格式 | 改成 |
|--------|------|
| `1. 第一点... 2. 第二点...` | "有三点，第一……，第二……，第三……" |
| 表格 | "你告诉我两件事：一是X，二是Y" |
| 超过3项并列 | 拆成 2 轮，每轮最多 3 项 |

审查完所有台词后，确认："所有台词已通过语音友好检查。"

---

## 最终判决步骤（通常是最后一步）

对最后一步追加引导：

> "最后一步要交付什么？
> - 有没有汇总表格（把前几步结果整合）？
> - 通过/不通过各有什么具体建议？
> - '条件性通过'的条件是什么？
> - 如果用户施压要'通融'，agent 怎么回应？"

---

## 对话里嵌工具（代码/模板）

对每个需要用户自己计算/执行的步骤，追加引导：

> "这一步里，用户需要自己跑什么计算或工具吗？
>
> 如果有——把代码/模板**直接嵌进标准台词**，用户不需要离开对话就能用：
> - 计算公式 → 直接写进台词，带示例输入/输出
> - Python / Excel 代码 → 代码块嵌入台词，参数名用用户的实际策略命名
> - 填写模板 → 表格嵌入台词，字段带说明
>
> 原则：agent 不应该只告诉用户'去跑 X'——而是把 X 直接递给他。"

如果用户说"我不懂代码"：
> "可以用文字描述计算步骤代替代码块，但要具体到'用 A 除以 B，得到 C，如果 C > X 则通过'的粒度。"

```markdown
## 第二部分 · 对话剧本

### Step 1 · [名称]

**🎯 子目标：** ...

**✅ 关闭准则：** ...

**🗣 标准台词：**
> agent：「...」

**📌 Fallback：**
| 情况 | Agent 动作 |
|------|-----------|
| 模糊答案 | 第1轮：引导推算。第2轮仍模糊：触发保守上限申报 |
| 拒绝提供 | ... |

**〔越界场景〕**
> 用户：「...」
> agent：「...」

**📚 KB 引用：** `knowledge-base/0N_xxx.md`

---

### Step 2 · ...
[重复结构]

---

## 第三部分 · 知识库引用清单
> 由 /agent-dialogue 自动生成

| Step | 文件 | 用途 |
|------|------|------|

```

**如果选了 🧠 超强记忆，追加第四部分：**

```markdown
---

## 第四部分 · 记忆设计
> 由 /agent-dialogue 生成（记忆型 agent 专属）

### 记忆槽
| 字段名 | 类型 | 写入时机 | 读取时机 |
|--------|------|---------|---------|

### 遗忘策略
- 永久保留：[永久字段列表]
- 过期：[X 次会话后清除]
- 不保留：[隐私/临时信息]
```
