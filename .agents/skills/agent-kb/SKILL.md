# Agent KB · 知识库

把 agent 剧本里引用的"专业知识/数据/案例"，变成结构化的 KB 文件。

## 为什么需要 KB 文件

agent 的判断力来自两个地方：
1. **System Prompt**：告诉 agent 怎么思考（框架/流程/铁律）
2. **知识库**：给 agent 具体的数字、案例、模板（"行业基准是 X%"）

没有 KB，agent 只能给出通用判断，无法引用具体证据。

## 前置

读取 `agents/[名称]/agent-script.md` 中的第三部分（知识库引用清单）。
不存在则提示："请先说 'use agent-dialogue skill' 完成对话剧本。"

展示引用清单，告诉用户："我们逐条建立这些文件，每次只建一个。"

---

## 逐条引导

对每个 KB 引用项，依次：

### A：明确类型

> "这个文件需要什么类型？
> A. 数字/基准（行业标准、统计结果）
> B. 判断框架（分析某类问题的方法）
> C. 真实案例（正面或反面的具体例子）
> D. 计算模板（公式或代码）
> E. 填写模板（用户填写的表单结构）"

---

### B：收集原始内容

**A 类（数字/基准）：**
提供：来源（论文/数据库/实测）/ 具体数值 / 适用范围 / 解读

**B 类（判断框架）：**
提供：判断什么情况 / 输入是什么 / 判断步骤 / 输出结论

**C 类（真实案例）：**
提供：发生了什么 / 结果怎样 / 说明了什么 / 正面参考还是反面警示

**D 类（计算模板）：**
提供：公式/代码 / 参数说明 / 示例输入输出

**E 类（填写模板）：**
提供：字段列表 / 字段说明 / 示例

---

### C：格式化并写入文件

写入 `agents/[名称]/knowledge-base/0N_[主题].md`：

```markdown
# [主题名称]

**来源：** [原始来源]
**适用场景：** agent 在 Step N 引用这个
**最后更新：** [日期]

---

## 核心内容

[主要内容]

---

## 在对话中如何引用

> agent 引用示例：「根据 [来源]，[结论]——这意味着你的情况是 [具体判断]。」

---

## 延伸参考

[可选：补充资料]
```

---

## 内容不足时的处理

用户没有具体数字/来源：

> "没有数字，我们可以：
> A. 改成'引导用户自己提供数据'（agent 不做断言，只做框架）
> B. 标注'待补充'，先让 agent 跑起来，后续真实测试时补充
> C. 告诉我你的领域，我帮你找公开基准数字"

---

## 完成后

所有 KB 文件生成后：
1. 确认文件存入 `agents/[名称]/knowledge-base/`
2. 更新 `agent-script.md` 第三部分引用清单（确认文件名对应）
3. 提示用户：说 "use agent-polish skill" 开始打磨
