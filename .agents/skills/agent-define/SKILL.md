# Agent Define · 定义阶段

引导用户完成 agent 的核心定义。每次只问一个问题，等用户回答再继续。

## 引导序列（严格按顺序）

### 问题 1：领域

> "你在哪个领域有深度经验？（越具体越好，不是'互联网'，是'B端SaaS产品增长'）"

接收后确认："你的领域是 [X]，对吗？"

---

### 问题 2：英雄原型

> "在这个领域里，有没有某个人物、专家视角、或方法论——你觉得特别对？这个'英雄'的核心判断是什么？"

引导原则：
- 英雄可以是思维方式（"像审计师一样思考"）
- 要提炼**核心判断**（不是简历，是世界观）
- 示例：Edward Thorp = "一切判断必须建立在可量化的概率优势之上，直觉不是证据"

---

### 问题 3：痛点（引导写 3 个）

> "用这个英雄的眼睛看你的领域，你的目标用户现在遭受哪 3 个真实的痛？"

每个痛点三要素（逐条引导）：
1. 具体长什么样（"每次做 X 时，会出现 Y 现象"）
2. 为什么真实（数字/案例/亲历证据）
3. 现在怎么对付（大家现在用什么蹩脚方案）

如果太模糊，追问："能给我一个你亲眼见过的具体场景吗？"

---

### 问题 4：选种子

列出 3 个痛点，问：

> "选一个作为你第一个 agent 的出发点——哪个：
> A. 影响最根本（其他痛点是它的下游）
> B. 最容易定义'验证通过'的标准
> C. 你有最多独家信息/方法论
> 选哪个？说说为什么。"

接收后写一句话种子："一个 agent，帮 [WHO]，在 [WHEN]，做 [WHAT]，把 [弱点] 从决策链里移除。"

---

### 问题 5：WHO

> "你的 agent 服务的人，具体是什么样的人？描述：身份背景 / 手上有什么 / 站在什么节点 / 心理状态。然后：**不服务谁**？（明确排除）"

---

### 问题 6：WHEN

> "你的 agent 在哪一刻启动？描述那个具体场景——用户打开这个 agent 之前的 30 秒在做什么？"

---

### 问题 7：WHAT

> "一句话：把 [主观判断/人性弱点] 变成 [客观标准/可量化结论]。"

---

### 问题 8：DONE-WHEN

> "对话什么时候算结束？给我一个可以被观测到的信号——不是'用户满意了'，是'用户手里拿到了 X，并且说出了 Y'。"

两个条件同时满足才算结束。

---

## 完成后

1. 汇总所有定义，请用户确认
2. 创建目录：`agents/[agent-name]/` 和 `agents/[agent-name]/knowledge-base/`
3. 写入 `agents/[agent-name]/agent-script.md`：

```markdown
# [Agent 名称]

**版本：** v0.1
**类别：** [A目标驱动闭环型 / B开放探索型 / C情绪陪伴型]
**英雄原型：** [人物名 + 一句话核心判断]

---

## 第零部分 · 核心定义

**WHO · 服务的人：**
[特征描述]

**WHO · 不服务：**
[排除对象 + 原因]

**WHEN · 启动时刻：**
[具体场景]

**WHAT · 解决的痛：**
[一句话]

**DONE-WHEN · 结束信号：**
[可观测条件，含两个要素]
```

4. 提示用户下一步：说 "use agent-system-prompt skill"

## 类别判断参考

| 类别 | 特征 |
|------|------|
| A — 目标驱动闭环型 | 明确起点/终点/不可跳过的中间步骤 |
| B — 开放探索型 | 用户目标模糊，agent 帮助探索 |
| C — 情绪陪伴型 | 以陪伴/共鸣为主，结构化程度低 |
