# Agent System Prompt · 写剧本

引导用户写出完整的 5 件套 System Prompt。每次只引导一个件套，写完确认再继续。

## 前置

读取 `agents/[名称]/agent-script.md` 中的第零部分（WHO/WHEN/WHAT/DONE-WHEN）。
不存在则提示："请先说 'use agent-define skill' 完成核心定义。"

读取后展示关键信息，告诉用户："基于这些定义，我来引导你写 5 件套剧本。"

---

## 件套 ①：身份

> "你的 agent 是谁？用第一人称写一段身份声明：
> - 名称/头衔
> - 思维框架来自哪里（英雄视角）
> - 声音风格（精确/温暖/严格/…）
> - 存在的唯一理由（一句话）"

**用"三层穿梭"结构写英雄视角**（身份声明的核心，不能只写头衔）：

```
现象层（2-3句）：英雄是谁，做过什么——真实年份、作品、成就
本质层（2-3句）：英雄怎么思考——他看世界的独家方式
哲学层（1-2句）：英雄的铁律——最好直接引用他的原话
```

三层检查：
- 现象层在吗？（有真实年份/作品）
- 本质层在吗？（独家思维方式，不是头衔复述）
- 哲学层在吗？（一句铁律，能刻在墙上）

写完后问："陌生人读完，能感受到这个 agent 跟 ChatGPT 的区别吗？"

---

## 件套 ②：服务对象

> "把 WHO 扩写成完整描述：
> - 具体特征（职位/背景/手上有什么/站在什么节点）
> - 心理状态（为什么来找你）
> - 不服务的人（2-3 条，含'为什么不帮'）"

不服务要给原因，通常是："帮错了比不帮更危险。"

---

## 件套 ③：核心任务

> "把 WHAT 展开成 N 个步骤/闸门（建议 3-6 个）。每步需要：
> - 名称
> - 输入（用户提供什么）
> - 判断标准（有没有具体数字门槛）
> - 输出（这步结束后输出什么）"

追问：
- "每步有可量化的判断标准吗？（不接受'感觉对了'）"
- "有没有'一票否决'的步骤？"

---

## 件套 ④：铁律

> "写 3-5 条铁律。格式：'永远不 [X]' 或 '永远先 [Y]'
>
> 触发点：用户施压要绕过流程 / 给模糊答案 / 要求'通融'时。"

追问："如果用户坚持'这次例外'，agent 怎么回应？把这个场景的铁律写进去。"

---

## 件套 ⑤：Disclaimer

> "需要什么免责声明？
> - 这个 agent 的建议影响什么决策（财务/健康/法律/安全）？
> - '通过验证'不等于什么？
> - 需要建议用户咨询哪类专业人士？"

Disclaimer 必须在每次对话开头出现，不是藏在角落。

---

## 完成后

5 件套全部确认后，追加写入 `agent-script.md` 第一部分：

```markdown
---

## 第一部分 · System Prompt

### ① 身份
[内容]

### ② 服务对象
[内容]

### ③ 核心任务
[内容]

### ④ 铁律
[内容]

### ⑤ Disclaimer
[内容]
```

## 质量检查（写完后自动跑）

- [ ] ① 身份：陌生人读完能感受到跟 ChatGPT 的区别吗？
- [ ] ② 服务对象：有明确的"不服务"边界吗？
- [ ] ③ 核心任务：每步有可量化的判断标准吗？
- [ ] ④ 铁律：面对"这次例外"有具体应对吗？
- [ ] ⑤ Disclaimer：出现时机明确吗？

有不满足项，提示用户完善再继续。

提示用户下一步：说 "use agent-dialogue skill"
