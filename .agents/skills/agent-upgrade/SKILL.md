# Agent Upgrade · 升级已有 Agent

为已有 agent 添加 WukongDojo 平台能力。不从头重写，只在现有基础上精确补充。

## 前置：读取现有 agent

读取 `agent-script.md`，提取：
- Agent 名称 + 当前版本号
- 第零部分B（如存在）：当前已有的平台能力
- 第二部分：所有 Step 的台词（用于语音审查）
- 第四部分（如存在）：已有记忆设计

展示当前状态：

```
当前：[agent名称] v[X.X]
会话模式：单次 / 多次（记忆型）
已有能力：[无 / 列出已有]
```

如果文件不存在，提示："请先运行 `/agent-define` → `/agent-system-prompt` → `/agent-dialogue` 完成 agent 脚本。"

---

## 第一步：能力选择

> "要为这个 agent 升级哪些能力？（可多选）
>
> ☐ 语音输入——台词改为口语友好
> ☐ 文件上传——添加文件收集步骤
> ☐ 文件输出——添加结果文件输出
> ☐ 超强记忆——添加跨会话记忆（需会话模式 = 多次；如当前是单次，先确认是否要改）"

---

## 第二步：输出改动范围清单

分析现有 agent-script.md，生成改动范围清单。**不直接改文件**，先让用户确认：

```
📋 升级影响范围（[agent名称] v[X.X] → v[X.X+0.1×N]）

【语音输入】（如选）
  Step 1 · [步骤名]：台词有 [N] 处列表需改为口语段落
  Step 3 · [步骤名]：台词超过 3 个并列项，需拆分
  共 [N] 步需要语音审查

【文件上传】（如选）
  建议在 Step [N] · [步骤名] 前/替代位置加 📎 文件步骤
  需新增提取模板：[字段列表]

【文件输出】（如选）
  最后一步（Step [N]）需追加文件输出设计

【超强记忆】（如选，记忆型才能选）
  Step 1 前需追加：会话开始协议
  每步末尾需确认：记忆钩子（存什么字段）
  最后一步需追加：记忆更新步骤
  需新增：第四部分 · 记忆设计

【第零部分B】（如无则新增，如有则更新）
  追加/更新平台能力声明
```

展示完毕后询问："这个改动范围对吗？确认后逐项引导。"

---

## 第三步：逐项引导修改

用户确认改动范围后，逐项执行：

**语音输入改造：**
逐条展示需改的台词，引导用户口语化改写，通过语音友好检查后写入。

**文件上传改造：**
用 📎 文件步骤框架（同 `/agent-dialogue` 的 📎 文件步骤框架），引导填写并插入或替换对应 Step。

**文件输出改造：**
在最后一步末尾引导追加：
- 输出文件名
- 文件内容结构
- 台词："我为你生成了 [文件名]，包含 [X]，点击下载。"

**超强记忆改造：**
按顺序引导：
1. 会话开始协议（Step 1 前）
2. 每步记忆钩子（逐步确认存什么）
3. 最后一步记忆更新步骤
4. 第四部分 · 记忆设计（记忆槽 + 遗忘策略）

---

## 完成后

每完成一项能力，版本号 +0.1。

更新 `第零部分B · 平台能力设计`（如无则新增）。

提示："升级完成。建议运行 `/agent-polish` 用新能力场景做一轮打磨测试，再运行 `/agent-publish` 更新发布。"
